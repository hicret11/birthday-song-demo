# Sing My Birthday — Reimagined: архитектура и большой план сборки

_Ресёрч-based. 2026-07-07. Читать вместе с `FLOW-REDESIGN.md` и `reimagined-flow-prototype.html`._

---

## 0. Главный принцип: не сносить фундамент, снести фасад

«Снести и перестроить» — про **опыт и концепцию**, не про инфраструктуру.
Платёжный контур, генерация текста/музыки, хранилище, БД, paywall — это боевой,
проверенный на проде **скелет**. Его выбрасывать = сжечь месяцы и заново ловить
те же баги. Мы **оставляем спину, переписываем впечатление** и добавляем три
принципиально новых способности, которых у продукта нет:

1. **Магия толпы** — реалтайм-соавторство многих людей в одну песню.
2. **Каст** — голосовые звонки персонажей (AI) и живые исполнители.
3. **Театральный фронт** — премьера как кинематографичный момент.

Вывод по языку: **один язык — TypeScript, сверху донизу.** Next.js закрывает и
фронт, и бэк (API-роуты), Remotion-воркер тоже TS/Node. Нагрузка здесь — это
оркестрация I/O и вызовы AI-API, а не CPU-тяжёлые вычисления, поэтому вводить
второй язык (Go/Python) для маленькой команды — только замедлит. Единый TS =
общие типы фронт↔бэк, один найм, одна кодовая база.

## 1. Keep / Add / Remove

### ✅ Оставляем (работает, доказано на проде)
| Слой | Технология | Зачем оставить |
| --- | --- | --- |
| Язык | **TypeScript** везде | Один язык, общие типы, быстрый найм |
| Фронт+бэк | **Next.js 16 + React 19** | Рабочая база; переписывать = минус недели (ресёрч подтвердил) |
| Стили | **Tailwind 4** | Уже есть, быстрый UI |
| Тексты | **OpenAI Responses API** | Structured output, уже мигрировано |
| Музыка | **Suno API** | Ядро продукта |
| Платежи | **Stripe** | Боевой paywall + webhook |
| БД | **Supabase (Postgres)** | Уже есть; станет ещё важнее (см. §4) |
| Кэш/лимиты | **Upstash Redis/KV** | Rate-limit, spend-cap, share-store |
| Медиа | **Vercel Blob** | Уже перевели сюда (и воркер тоже) |
| Видео | **Remotion-воркер** | Премиум-караоке; уже почти деплой-реди |
| Почта | **Resend** · Ошибки: **Sentry** | Работают |

### ➕ Добавляем (новые способности)
| Потребность | Технология | Почему именно это (ресёрч) |
| --- | --- | --- |
| Реалтайм-соавторство + presence | **Supabase Realtime** (Broadcast + Presence) | Уже на Supabase; вклады и так пишем в Postgres. Ноль нового вендора |
| Большие «watch party» (позже) | **Cloudflare Durable Objects / PartyKit** | Дешёвый fan-out (20:1 на входящие WS); подключаем ТОЛЬКО если премьеры станут вирусными |
| Надёжная оркестрация пайплайна | **Inngest** | Durable-шаги, `waitForEvent` на webhook Suno (не жжём время функции опросом), без отдельного сервера, интеграция с Vercel |
| Голос персонажа | **ElevenLabs Agents + Twilio** | Реалтайм-голос ~75мс, нативная телефония, дёшево за звонок |
| Театральные анимации | **GSAP** (set-piece) + **Motion** (UI) | GSAP стал бесплатным (2025) вкл. все плагины; Motion — для React-микроанимаций |
| Аудио-реактив премьеры | **Web Audio `AnalyserNode` + Canvas 2D**; **canvas-confetti** | Просто, 60fps на телефоне; WebGL только для одного 3D-центрпис |

### ➖ Убираем / переосмысливаем
- **Форму как парадигму.** Заменяем на разговор с продюсером «Майло» (см. flow-док).
- **Загрузку фото у paywall** → переносим в момент ожидания как со-авторство.
- **ffmpeg премиум-аудиограмму 9:16** — уже отключена детектом (падает на Vercel);
  премиум-видео теперь = Remotion. Оставляем простое видео как фолбэк.
- **Мёртвые R2-переменные** из доков/энва (уже почистили в воркере).

## 2. Архитектура (обзор)

```
                      ┌─────────────────────────────┐
   Отправитель ─────▶ │  Next.js 16 (Vercel)        │  ← весь UI + API-роуты (TS)
   Именинник  ─────▶ │  · Server Components         │
   Каст-вклады ────▶ │  · /api/* (тонкие ручки)     │
                      └──────┬───────────┬───────────┘
                             │           │
             событие «создать│песню»     │ подписка на комнату
                             ▼           ▼
                    ┌────────────┐   ┌──────────────────┐
                    │  Inngest   │   │ Supabase Realtime│  presence + broadcast
                    │ (durable)  │   │  (crowd room)    │
                    └─────┬──────┘   └────────┬─────────┘
        step: lyrics(OpenAI)│                 │ вклады пишутся в
        step: music(Suno)  ─┤                 ▼
        step: render(Remotion worker)   ┌───────────────┐
        step: voice(ElevenLabs+Twilio) ─┤   Supabase    │  Postgres (single source of truth)
        step: email(Resend)             │  + Blob (медиа)│
                    └────────────────────┴───────────────┘
                             │
                       Stripe (оплата + доп. букинги каста)
```

Принцип: **Vercel держит приложение и тонкие ручки; всё «долгое» — в Inngest;
всё «живое» — в Supabase Realtime.** На Vercel НЕ поднимаем WebSocket-сервер
(функции эфемерны — ресёрч это отдельно подчёркивает).

## 3. Три новых подсистемы

### 3A. Магия толпы (реалтайм-соавторство)
**Идея:** секретная ссылка → круг именинника добавляет строчку/фото/голосовое →
всё видно живьём → Майло сводит вклады в одну песню.

**Как строим (лениво, без нового вендора):**
- Комната = запись `gift` в Postgres; вклады = строки в таблице `contributions`.
- **Presence** (Supabase) — «сейчас 3 человека добавляют» (аватары).
- **Broadcast** (Supabase, НЕ Postgres Changes) — новые вклады появляются мгновенно;
  Broadcast масштабируется на fan-out, Postgres Changes упирается в auth-проверку на
  каждого подписчика (ресёрч).
- Лёгкая идентичность вкладчика: анонимный токен в cookie + имя (без полноценного
  логина). Модерация вкладов через тот же OpenAI-moderation, что уже есть.
- Когда отправитель закрывает сбор → Inngest-джоб собирает вклады → OpenAI лепит
  цельный текст → Suno → премьера.
- **Порог:** пока премьеры камерные (десятки–сотни зрителей) — Supabase Pro тянет
  (лимит 500 конкурентных). Вирусная премьера → выносим reveal-комнату на
  Cloudflare Durable Objects.

### 3B. Каст: звонок персонажа (AI-голос)
**Технически готово:** ElevenLabs Agents + нативный Twilio → агент звонит,
называет по имени, коротко поздравляет разговором. Себестоимость 1–3 мин = центы.

**Юридические перила (обязательны, встроены в продукт):**
- Только **свои оригинальные персонажи** («волшебник Золтар») и **архетипы**
  («пират», «фея») — не чужие бренды.
- **Никаких клонов голосов звёзд / named-персонажей** без лицензии (ELVIS Act TN,
  CA AB 1836/2602, дела Midler/Waits — это иск-магнит).
- **Явная маркировка «AI-звонок»** (EU AI Act, авг. 2025) + согласие получателя
  (для детей — родителя).
- Оркестрация звонка — Inngest-шаг после оплаты; статус/ретраи как у рендера.

### 3C. Каст: живые исполнители (гитарист/косплеер)
**НЕ маркетплейс на старте.** Concierge-MVP в одном городе: вручную отобранные
исполнители, бронь через нашу форму, координация руками, managed-комиссия 15–30%.
Технически это просто: таблица `cast_bookings` + Stripe + письма + админка (уже есть
каркас admin). Автоматизируем после подтверждённого спроса. Риск — операционка и
ответственность, не код.

## 4. Изменения в данных и оплате

**Новые таблицы (Supabase):**
- `gifts` — расширяем текущую share-модель: добавляем `crowd_enabled`,
  `crowd_closes_at`, `director_name`, `relationship`.
- `contributions` — `gift_id`, `author_token`, `author_name`, `type`
  (line|photo|voice), `content_url|text`, `status` (pending|approved).
- `cast_bookings` — `gift_id`, `kind` (ai_call|live_musician|character),
  `status`, `scheduled_at`, `stripe_payment_id`, `fulfillment_notes`.

**Оплата (Stripe), новые сценарии:**
- **Occasion-based, не подписка** (урок Cameo: строить на дарении, не удержании).
- **Add-on букинги каста** — отдельные one-time платежи поверх песни.
- **Групповой сброс (позже)** — несколько людей скидываются на подарок
  (Stripe + учёт долей); мощный социальный крючок, но не для MVP.

## 5. Оркестрация пайплайна на Inngest (надёжность)

Сейчас пайплайн живёт в Vercel-функциях с лимитами времени и «best-effort»
фоновыми задачами. Переносим в Inngest как durable-воркфлоу:

```
event "gift/created"
 → step.run  "generate-lyrics"        (OpenAI, ретраи)
 → step.run  "submit-suno"            (отправляем задачу)
 → step.waitForEvent "suno.completed" (webhook, timeout 10м — НЕ опрос)
 → step.run  "highlight-cut"          (аудио-нарезка)
 → step.run  "request-remotion"       (премиум-видео)
 → step.run  "send-song-ready-email"  (Resend)
   [если оплачен каст] → step.run "trigger-voice-call" (ElevenLabs+Twilio)
```
Каждый шаг чекпоинтится: падение = ретраится только упавший шаг, успешные не
повторяются. Дашборд Inngest даёт наблюдаемость всей воронки. Отдельный
always-on сервер не нужен.

## 6. Большой план сборки (по фазам)

**Фаза 0 — стабилизация текущего (перед любым реворком) · ~несколько дней**
Довести до прод-готовности то, что уже есть: ротация токенов, Vercel Pro, Stripe
LIVE, деплой Remotion-воркера, проверка слайдшоу. _(См. `LAUNCH-READINESS-REPORT.md`,
`VIDEO-LAUNCH.md`.)_ Это даёт рабочую монетизацию, пока строим новое.

**Фаза 1 — театральный опыт (реворк фасада) · ~2–3 недели**
- Разговор вместо формы (продюсер Майло), признание отправителя, «кто ты для них».
- Перенос фото/воспоминания в окно ожидания как со-авторство.
- **Премьера-пик**: GSAP-занавес, аудио-реактив (AnalyserNode+Canvas), confetti,
  авторский титр. Финал на гордость (пик-энд).
- Соблюдаем mobile-60fps (только transform/opacity, `prefers-reduced-motion`,
  ленивая загрузка GSAP/визуалов).
- _Результат:_ тот самый прототип, но живой и продающий.

**Фаза 2 — оркестрация + магия толпы · ~3–4 недели**
- Внедряем **Inngest**, переносим пайплайн на durable-шаги.
- **Crowd rooms** на Supabase Realtime: presence, broadcast, таблица contributions,
  модерация, сведение вкладов в одну песню.
- _Результат:_ подарок «от целого круга» — ключевая эмоциональная фича.

**Фаза 3 — AI-каст (звонок персонажа) · ~2–4 недели**
- 3–4 оригинальных персонажа, скрипты, ElevenLabs+Twilio, маркировка AI, согласия.
- Букинг как add-on в Stripe, звонок как Inngest-шаг.
- _Результат:_ первый выход «за экран», высокий wow, чисто по праву.

**Фаза 4 — живой каст (concierge-MVP) · отдельный ручной трек**
- Один город, вручную, managed-комиссия, админка бронирований.
- _Результат:_ проверка спроса на реальные впечатления без построения маркетплейса.

## 7. Риски и на что смотреть
- **Реалтайм-лимиты Supabase** (500 конкурентных на Pro) — следим за размером
  премьер; вырастет — выносим на Durable Objects.
- **Inngest-executions** биллятся по шагам — держим воркфлоу компактным.
- **IP/право в касте** — только свои персонажи, маркировка AI, согласия. Строго.
- **Операционка живого каста** — неявки/качество/ответственность; поэтому concierge,
  а не маркетплейс.
- **Не переусложнить фронт** — WebGL только для одного центрпис; иначе телефоны
  просядут.
- **Единый TS** — не поддаваться соблазну второго языка ради «модности».

## 8. Команда/навыки
Для всего плана достаточно **TypeScript/React-разработчиков** + немного
DevOps на воркер/деплой. Ни Go, ни Python, ни отдельного бэкенд-стека не нужно —
это сознательное решение ради скорости маленькой команды.
